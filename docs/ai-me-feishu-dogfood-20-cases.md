# AI-ME 飞书 20 条真实狗粮验收矩阵

本文档固定本轮 20 条真实飞书消息的测试意图、预期 Tool Calling 和验收动作。消息必须从真实飞书用户发送给 AI-ME 机器人；Webhook 重放只用于可靠性测试，不计入 20 条真实消息。

## 统一验收口径

每条消息必须具备以下证据：

1. `ai_me_feishu_webhook_event` 有入站事件和验签结果；
2. `inbox_item` 与外层 `ai_me_approval` 已关联；
3. 模型调用有 `ai_me_run`，工具调用有 `ai_me_tool_call`；历史兼容样本若缺少旧版运行数据，必须在决策账本中明确标记“未记录模型”；
4. 通过或驳回审批后形成终态；
5. 对外发送不得重复，失败必须可重试或进入死信；
6. 完成 1-5 分质量复盘，记录 `accepted`、`needs_retry` 或 `wrong`；
7. 面板能看到模型、Token、成本、人工介入原因和关联工作项。

## 额外可靠性用例

这些用例不占 20 条真实消息名额：

- 用同一个飞书事件 ID 重放两次，只允许生成一个 inbox 和一个审批；
- 模拟飞书发送超时，验证自动重试使用同一个幂等 UUID；
- 连续失败达到上限后进入死信，人工恢复后只能成功发送一次；
- 模拟 DeepSeek 超时和非法 JSON，审批仍需保留保守草稿与错误证据；
- 模拟员工失败，验证重试、重新分配和人工接管均有轨迹。

## 完成条件

- 固定批次显示 `20/20`，后续新消息不会挤掉本轮槽位；
- 20 条均完成质量复盘，且失败和驳回样本保留真实分数；
- 对外重复发送数为 0；
- 所有失败都能自动恢复或人工恢复；
- 每条决策都能关联模型、Token、成本、审批、工具调用和工作项。
